About

A DAG of tasks,
not a chain of prompts

Most LLM orchestration frameworks treat planning as an afterthought — a linear chain of tool calls hoping nothing breaks. cuddlytoddly takes a different approach.

Tasks are decomposed into a directed acyclic graph. Dependencies are explicit. Progress is checkpointed. When something goes wrong, you resume from exactly where you left off.

It's not babysitting your model — it's giving it the structure it was missing.

Features

Everything the toddler
needs to walk steady

⬡

Task Graph Decomposition

Complex goals are broken into a DAG. Each node is a discrete, executable unit with declared dependencies.

✦

Human-Guided Reliability

Configurable checkpoints where humans review or intervene. Trust but verify — at whatever granularity you need.

◈

Multi-Backend Support

Plug in Anthropic, OpenAI, or any compatible API. The planning layer is model-agnostic by design.

↺

Crash & Resume

State is persisted after every node. Failures restart only the affected branch — not the whole plan.

◻

Extensible Skills

Add custom tool definitions and execution backends. Build the skill library your domain actually needs.

Quick Start

Up and running
in three lines

terminal 
# Install
pip install cuddlytoddly

# Set your key
export ANTHROPIC_API_KEY=sk-ant-...

# Run a task
cuddlytoddly "Write a market analysis for electric scooters"

Requires Python 3.10+. Works with any Anthropic-compatible API key.
